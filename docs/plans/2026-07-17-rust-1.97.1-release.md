# Rust 1.97.1 Rebuild Release Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Publish a patch release whose native artifacts are rebuilt with Rust 1.97.1, without changing Terminal Commander's API, behavior, or Rust 1.92 MSRV declaration.

**Architecture:** Treat the compiler correction as an isolated release-provenance change. Update every active developer, CI, and release pin together; rely on the existing Linux gate's pin-consistency assertion; then publish through the existing release-please path before beginning unrelated correctness work.

**Tech Stack:** Rust 1.97.1, Cargo, cargo-nextest, GitHub Actions, release-please, npm trusted publishing.

---

### Task 1: Synchronize the authoritative compiler pins

**Files:**

- Modify: `rust-toolchain.toml`
- Modify: `CONTRIBUTING.md`
- Modify: `RELEASE_CHECKLIST.md`
- Modify: `.github/workflows/_build-platform-binary.yml`
- Modify: `.github/workflows/deprecate-version.yml`
- Modify: `.github/workflows/npm-binary-build.yml`
- Modify: `.github/workflows/npm-bootstrap-publish.yml`
- Modify: `.github/workflows/release-please.yml`
- Modify: `.github/workflows/secret-health.yml`

**Step 1:** Replace each active Rust 1.95.0 build pin with 1.97.1.

**Step 2:** Preserve `workspace.package.rust-version = "1.92"`; this release does not raise MSRV.

**Step 3:** Search the active configuration surfaces again and verify no 1.95.0 build pin remains.

### Task 2: Verify the rebuilt source tree

**Step 1:** Confirm `rustc --version` and `cargo --version` resolve through the repository pin to 1.97.1.

**Step 2:** Run formatting and pin-consistency checks.

**Step 3:** Run the authoritative Windows and Linux gates from `CONTRIBUTING.md` section 6.1, keeping only the newest required run for each purpose.

**Step 4:** Review the complete diff and verify it contains no product behavior or version-number edits.

### Task 3: Publish the patch release

**Step 1:** Commit with a release-please-readable `fix(build)` subject and push `main`.

**Step 2:** Inventory active GitHub Actions and cancel only superseded duplicate runs.

**Step 3:** Merge the generated release-please PR after its required checks pass.

**Step 4:** Verify the GitHub release, root npm package, all native npm packages, and publishable Cargo crates report the new version.

### Task 4: Clean the milestone

**Step 1:** Stop only TC-owned verification jobs; preserve AAP-owned jobs.

**Step 2:** Remove TC-owned Cargo targets and stale disposable worktrees/merged branches after publication is verified.

**Step 3:** Confirm local `main` equals `origin/main`, the worktree is clean, and no redundant CI remains active.
