# TC 0.1.70 Dogfood Findings — 2026-07-02

Method: an agent (Claude Code) drove real tasks through the TC MCP facade
exclusively — Rust builds, git, node/npm, PTY REPL, file ops, registry
packs, subscriptions, and WSL — on Windows 11 against daemon v0.1.70
(policy profile DeveloperLocal, allow_shell=false). Every claim below was
reproduced live in that session, not derived from code reading.

## What worked well (keep)

- `run_and_watch` one-shot flow: cargo check (19s), git log, wsl.exe -e
  commands — signals clean, exit state honest, `compact:true` projection
  is exactly right for the common case.
- Quiet-command receipt: never-matching rules returned
  `lines_suppressed:5` plus the full tail. No silence.
- `command_output_tail` on finished jobs: raw lines, truncation-flagged.
  The right escape hatch for unknown output formats.
- Shell-interpreter denial error TEACHES the remedy (argv lane vs
  shell_exec + the gating cap). Best error message in the product.
- Zero-output honesty: a WSL cargo blocked silently on a flock held by
  another build; TC reported bytes_total:0, state:running — the truth.
- Kill propagation into WSL: `command_stop` of a wsl.exe job left no
  orphaned Linux-side processes (flock/bash gone from the distro).
- PTY (ConPTY) round-trip: node -i REPL echo/output/prompt all captured;
  stop returns final counters.
- File watch end-to-end: watch_start + file_write triggered the rule with
  a line-accurate pointer.
- `registry_test` dry-run and pack import ergonomics (single-call
  import+activate with explicit scope).
- `recover_hint` on the degraded wait was honest and actionable (health →
  poll status → do not re-run).

## Bugs found (fix)

1. PATHEXT: argv ["npm","--version"] → ProgramNotFound; absolute
   npm.cmd works. Bare-name resolution ignores PATHEXT, locking TC out of
   npm/npx/yarn/pnpm on Windows. FIXED 4f5e66d (resolution after the
   interpreter denylist; req.argv stays bare for audit/pack-hint).
2. Discovery lies about shell_exec: system_discover + omni_status matrix
   report available:true while policy caps.allow_shell=false and the call
   returns PolicyDenied. Sessions gate truthfully; shell_exec must too.
   FIXED 0d97d25 (cap-truthful catalogue + matrix reason field; also
   covers allow_session and allow_remote gating).
3. UTF-16LE garbling: `wsl.exe --list --verbose` output becomes
   NUL-interleaved garbage signals. FIXED 630945c (latched per-stream
   BOM/heuristic Utf16Decoder ahead of line split; needs a live daemon
   smoke once a rebuilt daemon is installed).
4. Stale subscription liveness: a pty_stop'd job still shows
   state:"running" in every subsequent sub_pull liveness[], while
   probe_status honestly returns UnknownProbe for the same probe.
   FIXED 9a966d6 (PTY arm now asks PtyRuntime::liveness; regression
   test proven fail->pass around the fix).
5. Adapter transport is load-sensitive and misclassifies failures.
   Three live occurrences, all while the machine was compile-loaded:
   (a) run_and_watch wait_ms=60000 degraded with the fixed "IPC error
   interrupted the wait" hint; (b) the same degrade on a 60s clippy
   watch that returned well before 60s elapsed; (c) `command
   action=wait` (bucket_wait) returned `daemon_unavailable` twice while
   `health` succeeded in between (daemon uptime continuous - it never
   died). Investigation (read-only trace) disproved the timeout-constant
   hypothesis on the run_and_watch path (waits are chunked to 1s slices;
   no 60s constant exists); the code defect is that both degraded arms
   swallow the real IpcError (`Err(_)`) and print a canned hint, so the
   true trigger was undiagnosable. FIXED in this round: the hint now
   appends the underlying code+message. FOLLOW-UP (backlog): harden the
   adapter transport under load - a transient connect/read failure to a
   healthy daemon must not surface as `daemon_unavailable`; retry once
   on transient failure before classifying, and align the IPC client
   deadline with the longest blocking daemon wait it forwards.
6. registry deactivate: schema doc says version "Omit for the latest
   stored version" but the call rejects omission; missing required
   fields are revealed ONE PER CALL (version, then scope) — three
   round-trips to learn one call shape. FIXED 0d97d25 (version now
   optional, resolves latest adapter-side). Field-at-a-time missing
   errors remain a general serde-facade behavior (backlog).
7. import_pack idempotency: re-importing an identical pack mints new
   identical versions (v2) instead of reporting them in `skipped`.
   NOT FIXED (backlog).
8. Inline-rule identity: two different inline rules (".+" vs
   "WATCH_HIT") emitted events with the IDENTICAL rule id
   rul_edf88f26d1ba5b2f9404da36a61e22ed. Root cause: auto id is
   positional ("inline-0") and the RuleId is UUIDv5 over that string —
   verified byte-for-byte. FIXED c64652e (matcher digest in the auto
   id).
9. Repo hygiene found by the round: bench_events_since.rs merged
   unformatted (breaks cargo fmt --all --check; fixed 63d8595) and 3
   security.rs tests assert Unix path-glob semantics ungated on Windows
   (gated cfg(unix), same class as 81b2396).

## Improvements (ranked by pain)

1. No discovery primitive: files facade is single-file only (no
   list/glob), and with allow_shell off + cmd denied there is NO way to
   list a directory through TC on Windows. Either a bounded `files
   action=list` or a curated safe-binary story.
2. Per-action param contract is invisible in the flat facade schema:
   sub_pull silently IGNORED timeout_ms (the field exists schema-wide;
   the action honors wait_ms). Unknown-for-action fields should be
   rejected or aliased, never silently dropped.
3. `wait`/`events` responses have no `compact` projection (run_and_watch
   has one); full event records are ~15 lines of plumbing per one-line
   summary. sub_pull additionally resends the full liveness[] of ALL
   sources on every pull.
4. event_context requires bucket_id although evt_ ids are globally
   unique — pure ceremony.
5. pty_stdin cannot wait for the signals it provokes; every REPL
   interaction is 2 calls (stdin, then wait). A wait_ms on pty_stdin
   returning fresh signals would halve PTY round-trips.
6. No bulk deactivate (pack-level or multi-rule) — deactivating the
   cargo pack took 6 calls.
7. file_write is whole-content only — no append mode; watch-trigger
   tests and log-append workflows rewrite the full file.
8. suggest_from_samples heuristics are thin: from 6 realistic npm/tsc
   lines it proposed only `error-prefix` and missed `npm ERR!` and
   `error TS\d+` shapes entirely (and defaulted the proposal to
   stream:stderr without evidence).
9. WSL path translation (E:\... ↔ /mnt/e/...) is left to the agent;
   a helper (or docs recipe) would prevent silent cwd mistakes.
10. Policy observation, needs a decision: wsl.exe -e bash -lc "..."
    smuggles a full shell through the argv lane (argv[0]=wsl.exe passes
    the denylist). If allow_shell=false is meant as a boundary, WSL
    punches through it; if WSL is considered a separate machine, document
    that stance.

## Environment notes

- Session facade unix-only gating on Windows reported truthfully
  (available:false, clear reason) — good.
- npm resolves once given the absolute .cmd path; Rust std's
  BatBadBut-safe .cmd spawning applies.
